# 代码结构与可复用性审查

审查日期：2026-06-15

## 结论

项目主体架构已经具备可扩展平台的雏形，后端核心包基本符合“领域模型、适配器、队列、运行时、仓储”分层思想。当前最大问题不是技术选型，而是局部模块体积过大：网页端 `Dashboard` 组件承担了过多业务模型逻辑，开发期 `server.mjs` 同时承担 HTTP 路由、任务执行、本地文件库和源站注册表职责。

本轮已先重构两个高收益位置：

- 将 Dashboard 中的标签解析、任务/阅读状态标签、任务重跑请求构造、文件库排序和阅读进度计算抽到 `apps/web/src/lib/dashboard-model.ts`。
- 将开发期 API 的源站注册表读取、校验、桥接脚本解析和公开 descriptor 派生抽到 `services/dev-api/source-registry.mjs`。

## 已符合的设计模式思想

| 位置 | 当前思想 | 评价 |
| --- | --- | --- |
| `packages/domain` | 领域模型 / 值对象 / 生命周期方法 | 任务状态、进度、输出和事件语义集中，适合作为后端与 worker 的共同语言。 |
| `packages/source-adapter` | Adapter + Registry | 源站能力由统一 trait 和 registry 承载，新增站点不应复制整套业务流。 |
| `config/source-adapters.json` | 配置驱动 / 开闭原则 | Fangliding 与 18comic 通过同一配置结构接入，后续站点优先增加配置和桥接脚本。 |
| `scripts/source_bridge_core.py` | 模板方法雏形 / 共享基础设施 | HTTP、重试、cookie/header、文件名、JSON 写入等公共爬虫能力已集中。 |
| `packages/task-queue` | Queue 抽象 | 当前内存队列可替换为 NATS JetStream，而调用方不需要重写业务分发。 |
| `packages/task-runtime` | Dispatcher / Runtime / Reporter | 任务消费、来源分发、状态上报边界清楚，适合独立 worker 和 API 内置 worker 共用。 |
| `services/api` | Repository + Publisher | 仓储和事件发布不是直接散落在路由里，未来 PostgreSQL、事件总线替换空间较大。 |

## 主要问题

| 位置 | 问题 | 本轮处理 |
| --- | --- | --- |
| `apps/web/src/components/dashboard.tsx` | 单文件过大，视图、状态和业务纯函数混杂，难以复用到移动端或其他页面。 | 已抽出 `dashboard-model.ts`。下一步应继续拆 `useTasksDashboard`、`useLibraryDashboard` 和若干面板组件。 |
| `services/dev-api/server.mjs` | 开发期服务承担职责过多，新增本地功能时容易继续膨胀。 | 已抽出 `source-registry.mjs`。下一步应拆 `library-service.mjs`、`task-service.mjs`、`http-routes.mjs`。 |
| 源站 Python 桥接脚本 | 站点解析已经分离，但站点级解析仍需更多契约测试。 | 保持 `source_bridge_core.py` 作为公共层，后续给每个站点增加离线 HTML fixture 测试。 |
| 前端组件层 | 设计样式和业务控件还集中在单页控制台，公开平台后维护成本会上升。 | 暂不强拆 UI，优先把可测试业务规则迁出，降低回归风险。 |

## 本轮重构收益

- tag 多词条解析变成可复用函数，搜索页、未来批量导入页、移动端表单可以共用。
- 阅读进度、最近阅读、完整度、排序、任务重跑等规则不再绑死在一个 React 组件里。
- source adapter 注册逻辑脱离 HTTP 服务，后续新增站点时 dev-api 不需要重复写注册表解析代码。
- `server.mjs` 的启动流程更接近“读取配置 -> 物化 adapter -> HTTP 分发”，更符合单一职责和开闭原则。

## 后续重构优先级

1. 拆前端 hooks：`useTaskConsole`、`useLibraryFilters`、`useLibraryReader`。
2. 拆前端 UI 子组件：任务表、文件库卡片网格、详情抽屉、阅读器。
3. 拆开发期 API 服务：任务服务、本地文件库服务、导出服务、路由层。
4. 给 `dashboard-model.ts` 和 `source-registry.mjs` 增加单元测试。
5. 给每个源站桥接脚本增加离线 HTML fixture 测试，减少真实网络波动对回归验证的影响。
